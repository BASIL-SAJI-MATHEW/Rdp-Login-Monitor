# Security Policy

## Supported Scope

This project focuses on:

- Windows Security Event Log monitoring
- RDP login detection
- brute-force and suspicious access alerting
- operational security telemetry

## Reporting a Vulnerability

If you identify a security issue in this repository:

1. Do not publish the issue publicly first.
2. Prepare a clear report with reproduction details, affected files, and impact.
3. Send the report to the repository owner, **Basil Saji Mathew (BSM)**, through a private communication channel.

## What to Include

- vulnerability summary
- attack scenario
- affected component
- proof of concept or log sample
- suggested remediation if available

## Response Goals

Best effort goals:

- acknowledge receipt promptly
- assess severity and scope
- patch confirmed issues in a responsible timeframe
