# Contributing

Thank you for your interest in improving this project.

## Contribution Principles

- keep the architecture modular and readable
- prefer security-conscious defaults
- preserve attribution to Basil Saji Mathew (BSM)
- write clear commit messages and practical documentation
- add or update tests when behavior changes

## Recommended Workflow

1. Fork or branch from the latest mainline.
2. Create a focused feature or fix branch.
3. Keep changes scoped and well documented.
4. Run tests and validation checks.
5. Open a pull request with a concise summary, impact notes, and verification steps.

## Coding Expectations

- follow PEP 8
- use type hints
- preserve clean architecture boundaries
- prefer structured logging over ad hoc print debugging
- avoid introducing hard-coded secrets or environment-specific values

## Documentation Expectations

- update `README.md` when user-facing behavior changes
- update `CHANGELOG.md` for meaningful releases
- update `SECURITY.md` when reporting or support guidance changes
